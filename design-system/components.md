# Components

This document defines core components and their recommended usage patterns.

## Buttons

**Variants**

- Primary: Pier Navy background, white text.
- Secondary: White background, Pier Navy border and text.
- Ghost: Transparent background, Lake Blue text.

**States**

- Default, hover (darken), active (pressed shadow), disabled (reduced opacity).

**Usage**

- Use Primary for the main action on a page or section.
- Use Secondary for supportive actions.
- Use Ghost for low-emphasis actions in dense layouts.

## Links

- Use Lake Blue for inline links.
- Underline on hover and focus for clarity.

## Navigation

- Sticky header with Pier Navy background and white text.
- Use a high-contrast CTA on the right for ticketing.

## Hero

- Full-width image or gradient background.
- Overline for category, large headline, supporting copy, CTA group.

## Cards

- Use soft shadows and 8-16px radius.
- Keep card content aligned and consistent in length.

## Badges and chips

- Badges for status and categories.
- Chips for filters with optional icon.

## Alerts

- Use status colors with subtle tinted backgrounds.
- Keep copy brief, include action only when needed.

## Form fields

- 44px minimum height, 16px horizontal padding.
- Labels above inputs, helper text below.
- Show error states with Pier Red and clear messaging.

## Tabs

- Use an underline indicator in Pier Navy.
- Keep tab labels short and scan-friendly.

## Accordion

- Use for FAQs and long-form content.
- Clear open/closed indicator (chevron).

## Footer

- Dark background with light text.
- Include visit information, social, and quick links.
